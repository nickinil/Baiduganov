## 1. Добавление записи (позитивные тесты)
- **Тест 1.1**  
  Ввод: дата = "31.12.2025", температура = "-5.5", описание = "Снежно", осадки = Да.  
  Ожидаемый результат: запись появляется в таблице, поля очищены, сообщение об успехе.

- **Тест 1.2**  
  Ввод: дата = "29.02.2024", температура = "0", описание = "Пасмурно", осадки = Нет.  
  Ожидаемый результат: запись добавлена (2024 – високосный).

## 2. Проверка валидации (негативные тесты)
- **Тест 2.1 – неверная дата**  
  Ввод: дата = "31-12-2025" (неверный разделитель).  
  Ожидаемый результат: ошибка "Дата должна быть в формате ДД.ММ.ГГГГ".

- **Тест 2.2 – нечисловая температура**  
  Ввод: температура = "холодно".  
  Ожидаемый результат: ошибка "Температура должна быть числом".

- **Тест 2.3 – пустое описание**  
  Ввод: описание = "" (пробелы).  
  Ожидаемый результат: ошибка "Описание не может быть пустым".

- **Тест 2.4 – несуществующая дата**  
  Ввод: дата = "30.02.2023".  
  Ожидаемый результат: ошибка формата даты (ValueError).

## 3. Фильтрация
- **Тест 3.1 – фильтр по дате**  
  Добавьте две записи с датами "01.01.2024" и "02.01.2024".  
  Установите фильтр даты "01.01.2024", нажмите "Применить".  
  Ожидаемый результат: в таблице только запись за 01.01.2024.

- **Тест 3.2 – фильтр по температуре**  
  Записи с температурами 5, 10, 15.  
  Установите "Темп. выше" = 10, примените.  
  Ожидаемый результат: отображаются записи с температурами 15 (строго больше 10).

- **Тест 3.3 – комбинированный фильтр**  
  Дата = "01.01.2024", Темп. выше = 0.  
  Ожидаемый результат: только записи за указанную дату и с температурой > 0.

- **Тест 3.4 – сброс фильтра**  
  После фильтрации нажать "Сбросить фильтр".  
  Ожидаемый результат: отображаются все записи.

## 4. Сохранение и загрузка JSON
- **Тест 4.1 – сохранение**  
  Добавьте несколько записей, нажмите "Сохранить в JSON".  
  Проверьте, что в папке проекта создан файл `weather_data.json` с корректными данными.

- **Тест 4.2 – загрузка при старте**  
  Закройте приложение, откройте снова.  
  Ожидаемый результат: ранее сохранённые записи отображаются в таблице.

- **Тест 4.3 – ручная загрузка**  
  Удалите записи из интерфейса (не из файла), нажмите "Загрузить из JSON".  
  Ожидаемый результат: записи восстанавливаются из файла.

## 5. Краевые случаи
- **Тест 5.1 – отрицательная температура**  
  Ввод: температура = "-30". Запись должна добавиться.
- **Тест 5.2 – большое количество записей**  
  Добавить 50+ записей, проверить прокрутку таблицы и фильтрацию.
